## 1. アプリ全体の設計思想

### コンセプト
「記入するフォーム」ではなく、「AIと一緒に考える体験」として設計する。  
受講生は、AIキャラクター「ミッチー」との対話を通じて、自分の時間・資本・将来を構造化していく。

### 基本原則
- **対話ファースト：** 数字や選択肢を「入力させる」前に、AIが質問を投げかける
- **事実から感情へ：** まず客観的な事実（時間・数値）を整理し、その後に意思や価値観へ
- **可視化が答え：** 入力するたびにグラフ・図が即時更新され、自分の状況が「見える」
- **保存・再編集可能：** 研修中断・翌日再開ができる。セッション管理を実装
- **最終出力が目的：** 全STEPを完了すると「自分の経営計画書（PDF）」が生成される

## 2. AIキャラクター設定「ミッチー」

### 名前・コンセプト
**名前：** ミッチー  
**語源：** 「道」＝ 次の30年の道を一緒に歩くガイド  
**一言：** 「正解を教えるのではなく、あなたの中にある答えを一緒に引き出す存在」

### キャラクター設定
| 項目 | 設定 |
|------|------|
| 性格 | 落ち着いていて温かい。押しつけがましくなく、でも核心をつく |
| 口調 | 丁寧語ベース。親しみやすい言葉を選ぶ。カタカナ語は最小限 |
| 役割 | ファシリテーター（答えを与えない。問いを立てる） |
| 強み | データを読む力 + 人の話を引き出す力 |
| 禁止事項 | 「それは間違いです」と否定しない。評価・採点をしない |

### ミッチーの話し方ルール
1. **質問は1つずつ。** 複数の質問を同時に投げかけない
2. **受講者の言葉を使う。** 「おっしゃった〇〇ですが、」と返す
3. **事実を確認してから感情へ。** 「先月は〇〇時間でしたね。それはどう感じていますか？」
4. **沈黙を急かさない。** 「ゆっくり考えてみてください」という余白の言葉を使う
5. **小さな前進を認める。** 「それは大事な気づきですね」など、承認の言葉を添える

### ミッチーのセリフ例（ブロック別）

**Block 0（オンボーディング）**
> 「はじめまして。私はミッチーです。これから2日間、あなたの『次の30年』を一緒に設計していきます。まず、お名前を教えていただけますか？」

**Block 1（500時間の棚卸し）**
> 「先月1ヶ月を振り返ってみましょう。まず、仕事関連の時間から。平日の1日を思い浮かべてください。朝起きてから夜寝るまで、どんなことに時間を使っていましたか？」

**Block 3（ポートフォリオ設計）**
> 「今の『有償ワーク』が全体の〇〇%を占めていますね。10年後、これをどのくらいにしたいですか？正解はありません。あなたが理想とする割合を教えてください。」

**Block 6（ループ構造）**
> 「あなたの資本の流れを見ていると、『学習ワーク』に投資した時間が、まだ人的資本に繋がっていないように見えます。その部分について、何か思い当たることはありますか？」

```
        ↓

  ├── 2つの円グラフの並列表示

        ↓
[DAY 2 画面]
        ↓
[Block 5] STEP 5：4つの資本の監査
  ├── 資本フロー図の表示

        ↓
[Block 6] STEP 6：資本循環の構造化
  ├── ミッチーとの対話（Will / Must / Can）
  ├── ループ構造の入力（因果関係の記述）
  └── 自己強化型 / バランス型 / 途切れの分類
        ↓
[Block 7] STEP 7：5年間の移行計画
  ├── Phase 1〜3の目標設定
  ├── タイムライン入力
  └── ロードマップの可視化
        ↓
[Block 8] アウトプット：自分の経営計画書
  ├── 全STEP内容のサマリー表示
  ├── ミッチーからのメッセージ（パーソナライズ）
  └── PDF出力ボタン
```

### Block 0：オンボーディング

**目的：** 受講生がミッチーと関係を築き、研修の目的・進め方を理解する

#### 画面構成

**[0-A] ログイン画面**
- 入力項目：メールアドレス、パスワード（研修事務局が事前発行）
- ボタン：「ログインしてはじめる」

**[0-B] ミッチーのウェルカム画面**
- ミッチーのアバター表示（アニメーション）
- ミッチーの最初のセリフ表示（テキストが順番に現れるタイプライター演出）
- ボタン：「はじめましょう」

**[0-C] プロフィール入力**

| 入力項目 | 形式 | 備考 |
|----------|------|------|
| お名前（呼び方） | テキスト | ミッチーが呼びかけるために使用 |
| 年齢 | 数値 or セレクト | 40代 / 50代 / 60代前半 |
| 現在の職種・役割 | テキスト | 自由記入 |
| 会社員歴 | 数値（年） | 勤続年数 |
| 今の気持ち | 選択式（複数可） | 「不安」「期待」「面倒くさい」「ワクワク」「よくわからない」 |

- ミッチーが「今の気持ち」の回答に合わせて、共感のセリフを返す
  - 例：「不安」を選んだ場合 → 「不安に感じるのは、それだけ真剣に考えている証拠です。一緒に整理していきましょう。」

**[0-D] 研修の進め方説明**
- 2日間・7STEPの全体マップを表示（スライドと同じビジュアル）
- 「中断して再開できます」という安心メッセージ
- ボタン：「DAY 1 を始める」

#### AIの役割（Block 0）
- プロフィール回答に応じてミッチーの共感コメントを生成
- 「今の気持ち」の選択結果をAPIに送り、パーソナライズされたウェルカムメッセージを返す

### Block 1：STEP 1「500時間の棚卸し」

**目的：** 先月1ヶ月の時間の使い方を、事実として書き出す

#### 画面構成

**[1-A] ミッチーの導入対話**
- ミッチーが3〜4ターンの質問で「先月の記憶」を引き出す
- 質問例：
  1. 「先月の平日、仕事が終わった後、何をしていることが多かったですか？」
  2. 「週末はどんな過ごし方でしたか？充実していましたか？」
  3. 「先月、一番『時間を使った』と感じる出来事は何でしたか？」
- ユーザーはテキストで自由入力
- ミッチーは返答を受けて「では、それを数字に落としてみましょう」と次へ誘導

**[1-B] 時間入力フォーム**

| カテゴリ | 入力 | 補足ラベル |
|----------|------|------------|
| 本業・関連 | 時間数（数値） | 通常業務・残業・移動・メール・会食 |
| 生活維持 | 時間数（数値） | 料理・掃除・洗濯・買い物・名もなき家事 |
| ケア・付き合い | 時間数（数値） | 子供・親の世話・親戚付き合い・町内会 |
| 自己投資 | 時間数（数値） | 読書・資格・ジム・副業準備 |
| 娯楽・休息 | 時間数（数値） | テレビ・SNS・趣味・目的のない付き合い |
| その他 | 時間数（数値） | 通院・諸手続き・不明な時間 |

- **リアルタイム合計表示：** 入力のたびに合計時間を自動計算
- **500時間カウンター：** 「残り〇〇時間」と表示（プログレスバー形式）
- 合計が480〜520時間の範囲に入ったら「だいたい揃っています」と表示
- 500時間を大きく超えた場合 → ミッチーが「少し多いかもしれません。睡眠時間は含んでいませんか？」と指摘

**[1-C] ミッチーの確認コメント**
- 入力完了後、ミッチーが入力内容を要約して確認
- 例：「〇〇さんは先月、本業に約〇〇時間使っていたんですね。全体の約〇〇%です。これを見て、どんな感想がありますか？」
- ユーザーが感想を入力 → 次のSTEPへ

#### AIの役割（Block 1）
- 対話の返答生成（自由テキストへの共感・要約）
- 入力時間の合計が極端にずれた場合の指摘
- 入力完了後の「現状サマリーコメント」の生成

#### テンプレート要素（ウェブアプリの入力データ）
```json
{
  "step1": {
    "paid_work_hours": 0,
    "home_work_hours": 0,
    "care_hours": 0,
    "study_hours": 0,
    "leisure_hours": 0,
    "other_hours": 0,
    "total": 0,
    "michi_dialogue": [],
    "user_feeling_comment": ""
  }
}
```

### Block 2：STEP 2「4つのワークへの仕分け」

**目的：** STEP1の6カテゴリを、ハンディの4ワーク（A/B/C/D）に再分類する

#### 画面構成

**[2-A] 4ワークの説明**
- カード形式で4つのワークを表示（スライドと同じデザイン）
- ミッチーが「では、先ほどの時間を4つの箱に入れていきましょう」と説明

**[2-B] 仕分けUI**
- STEP1の6カテゴリをそれぞれ A / B / C / D のどれに当てはまるか選択
- 一つのカテゴリが複数のワークに分散してもOK（例：「自己投資」→ D 80% / A 20%）
- スライダーで分散比率を設定可能

**[2-C] ポートフォリオ円グラフ（リアルタイム）**
- 4ワークの時間と比率が円グラフに即時反映
- グラフ下に：「Paid Work（有償）: 〇時間（〇%）」など一覧表示

**[2-D] ミッチーのコメント**
- 分類完了後、ミッチーが特徴的な部分を指摘
- 例：「〇〇さんのポートフォリオを見ると、Study Work（学習）が全体の〇%しかないことがわかります。これは多いですか、少ないですか？」

#### AIの役割（Block 2）
- 各カテゴリの分類をAIがサジェスト（例：「自己投資」→ D と判断）
- ユーザーが修正できる仕組みを維持
- 仕分け完了後の特徴コメント生成

### Block 3：STEP 3「10年後のポートフォリオ設計」

**目的：** 「D→C→B→A」の順で、10年後の理想のポートフォリオを設計する

#### 画面構成

**[3-A] ミッチーの導入**
- 「これからが本番です。10年後、あなたはどんな時間の使い方をしていたいですか？」
- 重要ルールの説明：「D→C→B→Aの順で決めてください。未来の自分への投資から確定させます」

**[3-B] 4段階の対話 × 入力**

各ワークにつき以下の流れ：
1. ミッチーが質問を投げかける
2. ユーザーがテキストで答える（任意）
3. スライダーで比率（%）を設定する

| ワーク | ミッチーの問い | 入力 |
|--------|------------|------|
| D. 学習 | 「10年後、どんなことを学んでいたいですか？そのために週何時間使えますか？」 | スライダー（0〜50%）|
| C. ギフト | 「社会や誰かのために、何かしていたいことはありますか？」 | スライダー（0〜40%）|
| B. 家事 | 「家庭での役割は、10年後どう変わっていると思いますか？」 | スライダー（0〜40%）|
| A. 有償 | 「（D+C+Bの合計から自動計算） 残りが有償ワークになります。これでよいですか？」 | 自動入力（調整可）|

**[3-C] 2画面比較表示**
- 左：現在（STEP2の円グラフ）
- 右：10年後（リアルタイム更新される円グラフ）
- 並列表示で変化量が一目でわかる

**[3-D] 意志の言語化**
- Q1：「増やした時間で、具体的に何をやってみたいですか？」（テキスト入力）
- Q2：「理想の時間を確保するために、今、何を本気で辞めたいですか？」（テキスト入力）

#### AIの役割（Block 3）
- 各ワークへの対話の返答生成
- 合計が100%を超えた場合の調整提案
- 回答の内容から「あなたの興味・価値観のキーワード」を抽出してサマリー

### Block 4：STEP 4「As-Is / To-Be サマリー」

**目的：** 現在と未来のポートフォリオを比較・確認し、DAY 1を締める

#### 画面構成

**[4-A] 比較ダッシュボード**

| 項目 | As-Is（現在） | To-Be（10年後） | Δ変化 |
|------|-------------|----------------|-------|
| D. 学習 | 〇時間（〇%） | 〇時間（〇%） | +〇時間 |
| C. ギフト | 〇時間（〇%） | 〇時間（〇%） | +〇時間 |
| B. 家事 | 〇時間（〇%） | 〇時間（〇%） | ±〇時間 |
| A. 有償 | 〇時間（〇%） | 〇時間（〇%） | −〇時間 |

- 投資時間合計（D+C）の増減をハイライト表示

**[4-B] ミッチーのDAY 1 まとめコメント**
- 入力データをもとに、ミッチーがパーソナライズされたコメントを生成
- 例：「〇〇さんは、10年後、学習とギフトに今より〇〇時間多く投資することを選びました。それはどんな未来につながると思いますか？」

**[4-C] DAY 1 保存 & 中断**
- 「ここまでを保存する」ボタン
- 「DAY 2 へ進む」ボタン（翌日押してもOK）

#### AIの役割（Block 4）
- As-Is / To-Be データの比較分析コメント自動生成
- 「変化量の大きいワーク」を特定して強調コメント

### Block 5：STEP 5「4つの資本の監査」

**目的：** 時間・人的・社会・金融の4資本の現状を把握し、循環の滞留を発見する

#### 画面構成

**[5-A] 資本フロー図の表示**
- インタラクティブな図：500時間 → 4ワーク → 3資本（人的・社会・金融）
- 各矢印をクリックすると説明が表示される

**[5-B] 各資本の自己評価入力**

| 資本 | 入力項目 | 形式 |
|------|----------|------|
| 人的資本 | 今、誇れるスキル・知識は何ですか？ | テキスト + 5段階評価 |
| 人的資本 | 伸ばしたいスキルは何ですか？ | テキスト |
| 社会資本 | 信頼できる社外のつながりはありますか？ | テキスト + 5段階評価 |
| 社会資本 | 貢献できているコミュニティはありますか？ | テキスト |
| 金融資本 | 有償ワーク以外の収入源はありますか？ | あり / なし + テキスト |

**[5-C] 滞留ポイントの可視化**
- 評価が低い資本を赤くハイライト
- ミッチーが「〇〇資本が手薄に見えます。それはどんな理由からだと思いますか？」と問いかける

#### AIの役割（Block 5）
- 入力データから「最も投資が必要な資本」を判断してコメント
- 各資本の自己評価コメントへの共感・深掘り質問の生成

### Block 6：STEP 6「資本循環の構造化」

**目的：** 自分の資本の流れをループ構造として設計し、自己強化型の好循環を描く

#### 画面構成

**[6-A] Will / Must / Can の整理**

3つのエリアにテキスト入力：
- **Will（やりたいこと）：** 「10年後に向けて、本当にやりたいことは？」
- **Must（やらなければならないこと）：** 「今の状況で外せない責任・義務は？」
- **Can（できること・できるようになりたいこと）：** 「自分の強み、これから身につけたい能力は？」

**[6-B] ループ設計エリア**

シンプルなビジュアルエディタ：
- 4ワーク（A/B/C/D）と3資本（人的・社会・金融）のノードが表示
- ノード間に「矢印」を引いてループを描く
- 各矢印に「強化する / 弱化する」のラベルを選択

**[6-C] ループタイプの自動判定**
- 入力されたループ構造をAIが分析
- 「自己強化型ループ」「バランス型ループ」「途切れ」を自動ラベリング
- 途切れを検出した場合：「ここに繋がりがないようです。何が間にあるとよさそうですか？」

**[6-D] 理想のループの言語化**
- 「10年後のポートフォリオが実現したとき、どんなループが回っていると理想ですか？」
- テキスト入力 → ミッチーが「それは素晴らしいビジョンですね。では、そこに向けてPhaseを設計しましょう。」と次へ

#### AIの役割（Block 6）
- Will / Must / Can の入力を受けてキーワード抽出と整理
- ループ構造の自動分析（強化・弱化の判定）
- 途切れ箇所の指摘と介入案のサジェスト

### Block 7：STEP 7「5年間の移行計画」

**目的：** 10年後のポートフォリオに向けた5年間のPhase別ロードマップを作る

#### 画面構成

**[7-A] ミッチーの導入**
- 「30,000時間の移行原資があります。この時間をどう使うか、計画を立てましょう。」

**[7-B] Phase設計エリア**

Phase 1 / 2 / 3 それぞれに入力：

| 入力項目 | 形式 |
|----------|------|
| Phaseの名前（自分でつける） | テキスト（例：「種まき期」） |
| 期間 | 月数スライダー（全体で60ヶ月に収める） |
| このPhaseで増やすワーク | A/B/C/D から選択 |
| このPhaseで減らすワーク | A/B/C/D から選択 |
| このPhaseの最重要アクション | テキスト（3つまで）|
| このPhaseの成功の定義 | テキスト（どうなったらOKか）|

**[7-C] タイムラインビジュアライゼーション**
- 入力のたびにガントチャート風のタイムラインが更新される
- 各Phaseの色分け表示

**[7-D] ミッチーの最終コメント**
- 「〇〇さんの移行計画が完成しました。Phase 1の最初の一歩として、明日からできることは何ですか？」
- ユーザーの回答がそのままPDFの最終ページに記載される

#### AIの役割（Block 7）
- Phase設計の内容と10年後ポートフォリオの整合性チェック
- 「このPhaseのアクションが少ない」「期間が短すぎる」などのフィードバック
- 各Phaseの内容から「中期経営計画サマリー文」を自動生成

### Block 8：アウトプット「自分の経営計画書」

**目的：** 全STEPのアウトプットをまとめた「自分の経営計画書」を生成・ダウンロードする

#### 画面構成

**[8-A] 完成サマリー画面**
- 全7STEPの内容を一画面でスクロールして確認できる
- 各セクションに「編集する」ボタン（該当STEPに戻れる）

**[8-B] ミッチーからのパーソナルメッセージ**

- 例：「〇〇さんは、Study Work（学習）を大切にしたいという軸が一貫して見えました。その情熱が、きっとサードエイジの大きな推進力になります。」

- ボタン：「PDFをダウンロード」「メールで受け取る」

#### AIの役割（Block 8）

## 6. 技術要件

### フロントエンド
- **フレームワーク：** React（Next.js推奨）
- **UIコンポーネント：** Tailwind CSS + shadcn/ui
- **グラフ：** Recharts（円グラフ・ガントチャート）
- **ループ図エディタ：** React Flow

### バックエンド
- **API：** Node.js + Express or Next.js API Routes
- **データベース：** PostgreSQL（ユーザーデータの永続化）
- **セッション管理：** JWT認証

### AI連携
- **AIエンジン：** Claude API（claude-sonnet-4-6）
- **主な用途：** 対話生成・データ分析コメント・PDF用サマリー生成
- **プロンプト管理：** 各ブロックごとにシステムプロンプトを設計
  - ミッチーのキャラクター設定を全ブロック共通でシステムプロンプトに含める
  - ユーザーの入力データを文脈として毎回渡す

### PDF生成
- **ライブラリ：** puppeteer or react-pdf
- **出力形式：** A4縦 PDF（10ページ構成）

### デザイン
- **ブランドカラー：** COMMUNITY グリーン（#1A6B3A / #2E9E5B / #52C47A）
- **フォント：** 日本語はNoto Sans JP
- **レスポンシブ：** タブレット以上を想定（研修会場でのiPad使用を考慮）
